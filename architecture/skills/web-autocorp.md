# Web Corp Auto-Launch Protocol

## Когда срабатывает (триггеры)

Это skill срабатывает АВТОМАТИЧЕСКИ при любом из паттернов:

### Русские триггеры
- "сделай сайт", "создай сайт", "нужен сайт"
- "сделай лендинг", "создай лендинг", "лендинг для"
- "сделай интернет-магазин", "нужен магазин", "онлайн-магазин"
- "сделай многостраничник", "корпоративный сайт"
- "сайт для [бизнес/продукт]", "веб-сайт"
- "портфолио сайт", "сайт-визитка"
- "нужна страница", "создай страницу"

### English триггеры
- "build a website", "create a website", "make a landing page"
- "need a website", "build landing", "ecommerce site"
- "web development", "build a site"

### Kwork заказы (автоматически)
- Категория заказа: "Сайт", "Лендинг", "Магазин", "Веб"
- Ключевые слова в заказе: landing, website, интернет-магазин, лендинг

## Действие при срабатывании

```
1. Извлечь из запроса:
   - Тип сайта (лендинг / магазин / корпоративный / ...)
   - Продукт или бизнес
   - Дополнительные требования

2. АВТОМАТИЧЕСКИ запустить web-ceo с полным брифом

3. НЕ спрашивать пользователя — сразу в работу
```

## Шаблон авто-запуска

При обнаружении триггера немедленно запускаю Agent с web-ceo:

```
Тип: [определённый тип сайта]
Продукт: [извлечённый продукт/бизнес]
Требования: [все детали из запроса]
Источник: [user / kwork-order]
```

## Kwork интеграция

Kwork бот при получении заказа категории "веб" → передаёт в web-ceo:
```python
WEB_KEYWORDS = [
    "сайт", "лендинг", "landing", "website", "магазин",
    "shop", "интернет-магазин", "ecommerce", "визитка",
    "многостраничник", "корпоративный", "портфолио"
]
```
